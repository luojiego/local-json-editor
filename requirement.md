
# 📋 JSON 配置文件编辑器 - 产品需求文档 (PRD)

## 一、项目概述

### 1.1 项目背景
游戏开发过程中，策划和程序需要频繁编辑 JSON 配置表（如角色属性、道具数据、关卡配置等）。传统文本编辑器虽然功能强大，但存在以下痛点：
- 需要安装软件，团队成员环境不统一
- 缺少针对游戏配置的专用功能
- 非技术人员使用门槛高
- 无法快速定位和批量操作

### 1.2 产品定位
**基于浏览器的专业 JSON 配置编辑器**，通过 File System Access API 直接读写本地文件，提供：
- 零安装、跨平台、团队协作友好
- 游戏配置专用的增强功能
- 友好的视觉体验和交互设计

### 1.3 目标用户
- **主要用户**：游戏策划、数值设计师（非技术背景）
- **次要用户**：程序员（需要快速编辑配置）
- **使用场景**：日常配置调整、批量数据修改、配置表校验

### 2.2 快速搜索功能 ⭐⭐⭐⭐⭐

```yaml
需求描述: 输入关键词快速定位目标文件
```

## 四、非功能需求

### 4.1 性能要求
```yaml
文件扫描: 
  - 1000 个文件 < 1 秒
  - 10000 个文件 < 3 秒

文件打开:
  - 小文件（<100KB）< 100ms
  - 大文件（1-10MB）< 500ms
  - 超大文件（>10MB）显示加载进度条

编辑响应:
  - 输入延迟 < 16ms（60fps）
  - 语法高亮更新 < 50ms

内存占用:
  - 单个文件 < 10MB
  - 同时打开 10 个文件 < 100MB
```

### 4.2 兼容性
```yaml
浏览器:
  - ✅ Chrome 86+
  - ✅ Edge 86+
  - ❌ Firefox（File System Access API 未支持）
  - ❌ Safari（同上）

操作系统:
  - ✅ Windows 10/11
  - ✅ macOS 11+
  - ✅ Linux（Chrome 浏览器）
```

### 4.3 安全性
```yaml
数据隐私:
  - 所有文件读写在本地完成
  - 不上传任何数据到服务器
  - 不使用第三方分析工具

权限管理:
  - 用户手动授权目录访问
  - 可随时撤销权限
  - 刷新页面后需重新授权（可选记住权限）

错误处理:
  - 文件读取失败：显示错误提示
  - 权限被拒绝：引导用户重新授权
  - JSON 解析失败：显示原始文本 + 错误信息
```

### 5.2 关键技术点
```typescript
// 1. 目录扫描（递归读取所有 JSON）
async function scanDirectory(dirHandle, path = '') {
  const files = [];
  for await (const entry of dirHandle.values()) {
    if (entry.kind === 'file' && entry.name.endsWith('.json')) {
      files.push({ name: entry.name, path, handle: entry });
    } else if (entry.kind === 'directory') {
      const subFiles = await scanDirectory(entry, `${path}/${entry.name}`);
      files.push(...subFiles);
    }
  }
  return files;
}

// 2. 文件读写
async function readFile(fileHandle) {
  const file = await fileHandle.getFile();
  return await file.text();
}

async function saveFile(fileHandle, content) {
  const writable = await fileHandle.createWritable();
  await writable.write(content);
  await writable.close();
}

// 3. 模糊搜索
const fuse = new Fuse(files, {
  keys: ['name', 'path'],
  threshold: 0.3,
  includeMatches: true  // 高亮匹配部分
});
```
